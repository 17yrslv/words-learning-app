# Инструкция по тестированию системы пространств

## Что было исправлено:

Исправлена проблема с вложенными `collect`, которая блокировала обновление списка пространств в реальном времени.

## Как работает система:

1. **Автоматическое обновление списка пространств:**
   - `repository.getAllSpaces()` возвращает `Flow<List<Space>>`
   - Room автоматически эмитит новые значения при изменениях в таблице `spaces`
   - HomeViewModel подписывается на этот Flow и обновляет UI

2. **Создание нового пространства:**
   - Пользователь нажимает "+ Создать новое пространство"
   - Вводит название (например, "Italian")
   - Нажимает "Создать"
   - Новое пространство добавляется в базу данных
   - Room автоматически эмитит обновленный список
   - UI обновляется мгновенно
   - Приложение автоматически переключается на новое пространство

3. **Переключение между пространствами:**
   - Нажмите на кнопку с названием текущего пространства (слева сверху)
   - Выберите пространство из списка
   - Статистика и все данные обновятся для выбранного пространства

## Как протестировать:

1. Запустите приложение
2. Нажмите на кнопку "English" в левом верхнем углу
3. Нажмите "+ Создать новое пространство"
4. Введите "Italian" и нажмите "Создать"
5. Новое пространство должно сразу появиться в списке
6. Приложение автоматически переключится на "Italian"
7. Добавьте несколько итальянских слов
8. Переключитесь обратно на "English" - увидите английские слова
9. Переключитесь на "Italian" - увидите итальянские слова

## Технические детали:

- Используется Room Flow для реактивного обновления
- Два независимых корутины: одна для списка пространств, другая для текущего пространства
- Нет необходимости перезапускать приложение
- Все изменения отображаются мгновенно
